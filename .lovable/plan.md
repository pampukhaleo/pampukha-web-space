## 2. Редизайн портфолио в masonry

Файл: `src/components/PortfolioSection.tsx`.

Подход: CSS multi-column masonry (`columns-1 md:columns-2 lg:columns-3`, `gap-6`, элементы `break-inside-avoid mb-6`). Это даст Pinterest-эффект: разная высота карточек, плотная вертикальная упаковка.

Что меняется визуально:
- Убрать «Featured project» (большой верхний блок).
- Все 6 проектов — равноправные карточки в masonry.
- Разная высота за счёт реальных пропорций изображений (часть в `aspect-[4/3]`, часть `aspect-[3/4]`, часть `aspect-[16/10]`) — задаётся в данных проекта.
- Заголовок и описание — всегда видны под изображением (не только на hover), чтобы выглядело как полноценная карточка-плитка.
- Тонкая граница + лёгкий hover (рамка primary, scale 1.01) — в духе Linear.app.
- Клик по карточке открывает существующий `ProjectPopup` (без изменений).

Адаптив:
- mobile: 1 колонка
- md (≥768): 2 колонки
- lg (≥1024): 3 колонки

## Технические детали

- Никаких новых зависимостей.
- Masonry — чистый Tailwind через CSS columns; не вводим `react-masonry-css` (избыточно).
- Технологии проекта в данных `portfolioItems` заменяются на нейтральные ярлыки (отображение тегов — если нужно — внутри попапа, который уже умеет их рендерить).
- Все строки идут через `t(...)` — никаких хардкод-текстов.
- Проверка: `npm run build` пройдёт автоматически; визуально проверю secciones Hero, Services, Portfolio.
